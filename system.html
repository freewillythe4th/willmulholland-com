How it runs

The loop, stage by stage

The brain is always listening, so launch day starts at ninety percent. When a feature ships, every asset comes out and the whole company gets armed. And it does not stop at ship: adoption and measurement feed back in, shaping the next launch and what gets built next.

00
Intelligence in
The senses. Everything that tells you what to say, landing in one place instead of twenty tabs.
Know exactly how buyers describe the problemGong sales calls · reviews on G2, Capterra, App Store and Google Play (soon)

Positioning and messaging get written from how buyers talk, the language, the objections, the reasons deals are won and lost, pulled from real calls and reviews instead of from guesses.

See what every competitor is doing, liveTheir ads on Google, LinkedIn and Meta · their web pages · screen recordings of their product · a daily file on 20 competitors

A rival's whole strategy, read off what they ship and spend on, and already current the moment it is needed instead of a fresh research project each time.

Nothing the team learns gets lostInternal meetings, captured through Granola and Notion

Every product, sales and strategy conversation is captured and routed in, so it lives in the system instead of in one person's memory.

Every brief fully scoped before buildingA structured interview that captures the full plan first

The work starts from a clear, complete brief, with nothing important left assumed.

01
Memory + context
Memory and retrieval. One context layer that reads all of it at once. The moat is here, not the model.
On-brand, approved, and differentiated messagingEncoded brand voice · approved-copy library · the differentiated positioning

Every draft comes out in the brand voice, consistent with the copy already signed off, and carrying the differentiated positioning, not the generic line any competitor could write.

Every project and decision stays up to dateOperational reference rebuilt twice daily: priorities, people, products, projects, decisions, metrics

One reference covering the live priorities, the people, the products and projects, the decisions made and why, and the latest metrics is rebuilt twice a day from the latest meetings, so the work never runs off stale context. Each skill also keeps its own notes current, so the system sharpens over time instead of starting cold.

Everything ever learned is one search awayOne organised knowledge base · context loads itself for the task

Every project, person, decision and source lives in one place the system reads in full, with the relevant context pulled into whatever is being worked on.

02
Synthesis
The PMM craft chain. Intelligence becomes the launch, one ordered pass per feature.
research→ audience→ whitespace→ positioning→ messaging→ pricing→ GTM plan→ launch plan→ enablement
Gates over every draft: buyer-persona test · humaniser · fact-checker · refine-spec · council on high-stakes calls.
how the gates keep the bar high
Buyer-persona test. Every line checked against the real buyer's worldview and ban list before it ships.
Humaniser. Strips AI tells, kills the slop patterns, tightens to spoken rhythm.
Fact-checker. Claims verified against the source before they go out.
refine-spec. Pressure-tests the plan before anything gets built.
Council. A panel of reviewer agents that argues the high-stakes calls from different angles.
03
Outputs
One approved messaging doc fans out to every launch asset, in one voice.
GTM plan Sales one-pager Website updates Help article Blog Founder post Company-page post Messaging doc Lifecycle emails Battlecard Video walkthrough Asset for everyone to post
The engine that already works: the content-refresh skill turns one approved messaging doc into four derived assets in a single pass, with provenance on every claim and an approval gate before anything ships. Generalising that fan-out to the full list is build 01 on the roadmap.
04
Distribution
Arm every team. Each person handed a ready-to-post asset, staged to approve then publish on launch day.
Marketing Sales Product Lifecycle Social Founder Creator team
The rails: social posts staged on LinkedIn, X and YouTube for one-click approval · lifecycle email through the email platform · site updates straight to the website · collateral into the shared drive · internal amplification through Slack. Nothing publishes without a human approving it.
05
Adoption + lifecycle
The launch is not done when it ships. It is done when people use the thing. Move the user from seeing it to adopting it.
Lifecycle email journeys Welcome + onboarding series Activation tutorials · YouTube + in-app Personalised by segment In-app triggers Paid-feature adoption nudges Expansion + retention prompts
The in-app layer: partner with lifecycle and product to design the moment a user starts using a paid feature. Running on email today, the personalised in-app trigger is the next build. The content layer: activation tutorial videos, built from the questions sales and customers actually ask, run on YouTube and sit inside the product to pull users to first use.
06
Measurement
Close the loop. Did it land. What moved adoption, what fell flat, and what the next launch should do differently.
Win/loss capture Launch retro Signal back into the brain Feeds the product roadmap Adoption + usage metrics Campaign-level attribution
The honest gap: today the loop closes by hand, a retro written back into the intelligence layer. Automating the metrics-to-brain feedback is build 03.

Who runs it

Agents run the system. One operator directs it.

A manager agent runs a stack of worker lanes, one lane per task, each with the right skill loaded and its own memory of the work so far. Six primitives keep it honest.

Goals

A finish line it cannot fake

Every task gets a written goal with a measurable end state. The agent cannot stop until the condition actually holds, with proof in the transcript. Done means verified done, not "sounded finished".

Loops

Recurring work that paces itself

Work that repeats, like intel sweeps, monitors and content refreshes, runs as a loop that decides when to check back. The system keeps working between conversations, not just during them.

Workflows

Fan-out with a referee

When a job is genuinely parallel, like a research sweep or a review panel, it fans out to multiple agents at once, then adversarial verifiers attack the findings before anything is trusted.

Hooks

Every request routes itself

Before the agent writes a word, hooks load the right project context, pin the right skill, and route every file to where it belongs. Nothing depends on remembering to set things up.

Compound rules

The same mistake never twice

Every correction becomes a standing rule that loads into every future session. Feedback given once changes how the whole system works from then on.

Skill memory

Every run starts where the last ended

Each skill keeps its own log of decisions, files and learnings, synced automatically at the end of every session. The system sharpens with use instead of starting cold.
