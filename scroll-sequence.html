Scroll-driven image sequence.

The Apple-product-page effect: frames swap as you scroll, so a sequence of images plays under the reader's control. This demo runs on synthetic frames so the mechanism works with no image files. Swap in real JPEGs and it becomes the real thing.

Make it real in three steps.

Drop in your own frames

  1. Export your animation as a numbered JPEG sequence: frame_0001.jpg to frame_00NN.jpg. Put them in images/sequences/<name>/.
  2. In this file's CONFIG block, set useRealFrames: true, set frameCount to how many frames you have, and point framePath at your folder.
  3. Drop the .seq section and its script into any page. Adjust the section height (400vh) for faster or slower playback.

Frames load lazily and draw on a canvas at device resolution, so it stays sharp and smooth. Keep each JPEG light (around 1280px wide, quality 70-80) so a 60-80 frame sequence stays fast. Honest trade-off: this is the same technique Framer's Workshop plugin generates, just living in your own code with no Framer account or monthly cost. If you would rather drive it inside Framer, that route is in the handoff notes.
